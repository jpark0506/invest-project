# mvp.md — MVP 범위 축소/제외 목록(우선순위 포함)

## 0. 목표

- "다음 달에도 쓰게 만드는가"를 가장 빠르게 검증한다.
- 반드시 필요한 기능만 남기고, 나머지는 v2 이후로 미룬다.

## 1. MVP Must-have (절대 포함)

1. 로그인(이메일 매직링크)
2. 포트폴리오 설정(종목/비중) + 저장
3. 플랜 설정(월 예산, 매수일, 2~3차 분할) + 저장
4. 주문표(Execution) 조회
    - 현재가
    - 목표금액
    - 권장 주수
    - 이월금
5. 스케줄러가 매수일에 주문표를 생성
6. 알림 발송(메일 1개 채널이면 충분)
7. 주문 완료 체크(Confirm) + 간단 메모

## 2. MVP Should-have (있으면 좋지만 2순위)

1. price_snapshots 캐시(재현성/비용 절감)
2. 알림 채널 추가(텔레그램)
3. 간단한 신호 라벨(과열/중립/완화) — 점수 없이도 가능
4. 실행 실패/오류 관리 화면(관리자용 최소)

## 3. MVP Won't-have (v2로 이관)

### AI/분석

- LLM 코멘트(자동 생성) → "사용자 클릭 시"로 미루기 가능
- 복합 지표(RSI+MA+52주) 완성 → 단일 지표/라벨로 시작
- 전문가 의견(Strong Buy 등) → 데이터 소싱/법적 표현 이슈로 후순위
- 백테스트/시뮬레이션/수익률 예측 → 절대 MVP 제외

### UX/플랫폼

- 실시간 차트/캔들 → 제외
- 푸시 알림(Web Push) → 제외 (메일/텔레그램으로 대체)
- 오프라인 완성형 PWA → 제외 (manifest만)
- 다계좌/다통화 완전 지원 → 제외 (KRW 국내 먼저)

### 실행/연동

- 자동주문/증권사 연동 → 제외(법/복잡성 증가)
- 체결가 자동 수집 → 제외

## 4. "축소해도 되는" 신호

- 개발 일정이 늘어지면 아래부터 제거:
    1. AI 코멘트
    2. 과열 점수(정밀) → 라벨만
    3. 텔레그램 알림
    4. price_snapshots(캐시)
- 그래도 남아야 하는 것은:
    - 주문표 계산 + 알림 + 확인(Confirm)

## 5. MVP 성공 기준(재확인)

- 매수일 알림을 받고 → 주문표 확인 → 실제 주문 → Confirm 체크
- 다음 달에도 동일 행동이 반복되는지(리텐션)
